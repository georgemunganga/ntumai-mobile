# Quick Start Guide\n\n## For Frontend Developers\n\n### 1. Setup (5 minutes)\n\n```bash\n# Install dependencies\npnpm install\n\n# Start development server\npnpm start\n\n# Run on simulator\npnpm ios    # macOS only\npnpm android\n```\n\n### 2. Understanding the Project Structure\n\n- **`/app`** - All screens and pages\n- **`/src/store`** - State management (Zustand)\n- **`/src/api`** - API services and mock data\n- **`/src/components`** - Reusable components\n- **`/src/navigation`** - Navigation logic\n- **`/src/utils`** - Helper functions\n\n### 3. Making Changes\n\n#### Add a New Screen\n\n```typescript\n// app/(customer)/NewScreen.tsx\nimport React from 'react';\nimport { View, Text, TouchableOpacity } from 'react-native';\nimport { useRouter } from 'expo-router';\n\nexport default function NewScreen() {\n  const router = useRouter();\n  \n  return (\n    <View className=\"flex-1 bg-white\">\n      <Text className=\"text-2xl font-bold\">New Screen</Text>\n      <TouchableOpacity\n        onPress={() => router.back()}\n        className=\"bg-green-600 rounded-lg py-3 px-6\"\n      >\n        <Text className=\"text-white font-bold\">Go Back</Text>\n      </TouchableOpacity>\n    </View>\n  );\n}\n```\n\n#### Use Store Data\n\n```typescript\nimport { useOrderStore } from '@/src/store';\n\nfunction OrderList() {\n  const { orders, loading, fetchOrders } = useOrderStore();\n  \n  React.useEffect(() => {\n    fetchOrders();\n  }, []);\n  \n  if (loading) return <Text>Loading...</Text>;\n  \n  return (\n    <FlatList\n      data={orders}\n      renderItem={({ item }) => <OrderCard order={item} />}\n    />\n  );\n}\n```\n\n#### Navigate Between Screens\n\n```typescript\nimport { useCustomerNavigation } from '@/src/navigation/NavigationHelpers';\n\nfunction HomeScreen() {\n  const nav = useCustomerNavigation();\n  \n  return (\n    <TouchableOpacity onPress={() => nav.goMarketplace()}>\n      <Text>Go to Marketplace</Text>\n    </TouchableOpacity>\n  );\n}\n```\n\n### 4. Styling\n\nUse Tailwind CSS classes:\n\n```typescript\n// Color classes\nclassName=\"bg-green-600 text-white\"\n\n// Spacing\nclassName=\"px-6 py-4 mb-2\"\n\n// Layout\nclassName=\"flex-1 flex-row items-center justify-between\"\n\n// Text\nclassName=\"text-2xl font-bold text-gray-900\"\n```\n\n### 5. Common Tasks\n\n#### Fetch Data from API\n\n```typescript\nconst { vendors, loading, error, fetchVendors } = useMarketplaceStore();\n\nReact.useEffect(() => {\n  fetchVendors();\n}, []);\n```\n\n#### Update Store State\n\n```typescript\nconst { addToCart } = useCartStore();\n\nconst handleAddToCart = (product) => {\n  addToCart(product);\n};\n```\n\n#### Show Loading State\n\n```typescript\nif (loading) {\n  return <ActivityIndicator size=\"large\" color=\"#16A34A\" />;\n}\n```\n\n#### Handle Errors\n\n```typescript\nif (error) {\n  return (\n    <View className=\"flex-1 items-center justify-center\">\n      <Text className=\"text-red-600\">Error: {error.message}</Text>\n      <TouchableOpacity onPress={() => retry()}>\n        <Text className=\"text-blue-600 mt-4\">Retry</Text>\n      </TouchableOpacity>\n    </View>\n  );\n}\n```\n\n---\n\n## For Backend Developers\n\n### 1. Understanding the API Structure\n\nThe app uses mock API services in `/src/api/mockServices.ts`. To integrate your backend:\n\n1. Update endpoints in `/src/api/client.ts`\n2. Replace mock implementations with real API calls\n3. Implement authentication token management\n\n### 2. API Endpoints Needed\n\n#### Authentication\n- `POST /auth/login` - Login with phone\n- `POST /auth/verify-otp` - Verify OTP\n- `POST /auth/refresh` - Refresh token\n\n#### Marketplace\n- `GET /vendors` - List vendors\n- `GET /vendors/:id` - Get vendor detail\n- `GET /products` - List products\n\n#### Orders\n- `POST /orders` - Create order\n- `GET /orders` - List orders\n- `GET /orders/:id` - Get order detail\n- `PUT /orders/:id/status` - Update status\n\n#### More endpoints in `API_INTEGRATION_GUIDE.md`\n\n### 3. Response Format\n\nAll responses should follow this format:\n\n```json\n{\n  \"success\": true,\n  \"data\": { /* actual data */ },\n  \"message\": \"Success message\"\n}\n```\n\nError responses:\n\n```json\n{\n  \"success\": false,\n  \"error\": {\n    \"code\": \"ERROR_CODE\",\n    \"message\": \"Error message\"\n  }\n}\n```\n\n### 4. Authentication\n\nImplement token-based authentication:\n\n```typescript\n// Login response\n{\n  \"token\": \"jwt_token_here\",\n  \"refreshToken\": \"refresh_token_here\",\n  \"user\": {\n    \"id\": \"user_id\",\n    \"phone\": \"+1234567890\",\n    \"role\": \"customer\"\n  }\n}\n```\n\n### 5. Integration Steps\n\n1. **Update API Client**\n   ```typescript\n   // src/api/client.ts\n   const API_BASE_URL = 'https://your-api.com/api/v1';\n   ```\n\n2. **Replace Mock Services**\n   ```typescript\n   // src/api/mockServices.ts\n   export const authService = {\n     login: async (phone: string) => {\n       const response = await apiClient.post('/auth/login', { phone });\n       return response.data;\n     },\n   };\n   ```\n\n3. **Test Integration**\n   - Test each endpoint\n   - Verify data formats\n   - Test error scenarios\n\n### 6. Data Models\n\nKey entities the app expects:\n\n**User**\n```typescript\n{\n  id: string;\n  phone: string;\n  email: string;\n  firstName: string;\n  lastName: string;\n  avatar: string;\n  role: 'customer' | 'tasker' | 'vendor';\n  rating: number;\n  verified: boolean;\n}\n```\n\n**Vendor**\n```typescript\n{\n  id: string;\n  name: string;\n  logo: string;\n  rating: number;\n  deliveryTime: string;\n  deliveryFee: number;\n  minOrder: number;\n  categories: string[];\n}\n```\n\n**Product**\n```typescript\n{\n  id: string;\n  vendorId: string;\n  name: string;\n  description: string;\n  price: number;\n  image: string;\n  category: string;\n  inStock: boolean;\n}\n```\n\n**Order**\n```typescript\n{\n  id: string;\n  customerId: string;\n  vendorId: string;\n  items: OrderItem[];\n  totalAmount: number;\n  deliveryAddress: Address;\n  status: 'pending' | 'confirmed' | 'preparing' | 'ready' | 'on_way' | 'delivered';\n  driverId?: string;\n  createdAt: string;\n  updatedAt: string;\n}\n```\n\nSee `API_INTEGRATION_GUIDE.md` for complete data models.\n\n---\n\n## Useful Commands\n\n```bash\n# Development\npnpm start              #
